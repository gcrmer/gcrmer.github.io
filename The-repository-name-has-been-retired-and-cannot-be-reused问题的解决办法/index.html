The repository [name] has been retired and cannot be reused问题的解决办法

2022/04/07

背景:

我在 Github 上更改仓库名时出现 The repository [你的仓库名] has been retired and cannot be reused 这个问题。

解决方式 ①

一开始我是在 Google 上搜索，告诉我联系 Github Support 解决，我看到确实有人通过这种方式解决了问题，但是回复的不是很快(别问我为什么知道，因为我发过邮件，过了好几天回复的，但是态度挺好的，给 Github 点个赞),毕竟 Github 上的开发者太多了。

解决方式 ②:

我在无意之中发现了另一个解决办法，就是你先把用户名改成别的名字，然后再去修改你的仓库名，最后再将用户名改回来就可以了，这个问题就完美解决了，本来我以为我发现了新大陆，直到我在 StackOverflow 上看到早就有人回答了我这个方法，我怎么没早点发现，哈哈哈。

心得:

以后解决问题 Google 解决不了的，一定去 StackOverflow 上看看，切记切记.

意义:

为什么要写这篇博客呢？是因为我觉得有必要记录一下自己解决这个问题的过程，说实话问题是不分大小的，小的问题也可能困扰我们很长时间，还有在百度上我没有看到关于这个问题的解决方法，如果这篇博客能够帮助别人解决问题，我也是十分开心的。
