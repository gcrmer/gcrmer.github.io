三、项目遇到的问题
问题 1:
运行

之后，在本地可以正常预览但是部署到 github 之后就发现切换路径时 url 是错的, 而且一刷新页面就乱掉了，后来在网上 google 搜这个问题, 发现 githubpages 有二级域名, 但是我们编译生成的文件路径都是以/开头，搜索之后找到了解决的办法
解决方式:
①package.json 里加上
“homepage”：“./”
表示我们加载资源的时候用的是相对路径
② 把 BrowserRouter 换成 HashRouter
如果使用 BrowserRouter 是需要服务器作相应配置的，如果没做页面一刷新就会出问题，修改完成后在运行 yarn build 就会发现 index.html 里的文件路径都是以./开头

问题 2:
在使用 mobx 的 observable 后，编译失败，提示不能使用”@”符号
解决方式：
安装 babel/plugin-proposal-decorators
使用命令 yarn add @babel/plugin-proposal-decorators 进行安装,然后在 package.json 进行配置 此处会出现问题就是 create-react-app 把很多东西已经封装进去了，我们无法做出修改，在网上搜索得知这里需要运行 yarn eject 把配置释放出来，但是如果直接运行 yarn eject 会出错，因为我们的代码还没有提交，所以这里要先提交再运行 yarn eject，然后在 package.json 里的 babel 处加上
