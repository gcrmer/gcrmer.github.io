项目遇到的问题:
运行 yarn build 之后
cd build
运行 http-server 在本地可以正常预览但是部署到github之后就发现切换路径时url是错的, 而且一刷新页面就乱掉了
后来在网上google搜这个问题, 发现githubpages有二级域名, 但是我们编译生成的文件路径都是以/开头, 为了解决它
作出修改：
1.package.json里加上
“homepage”：“./”
表示我们加载资源的时候用的是相对路径

2.把BrowserRouter换成HashRouter
如果使用BrowserRouter是需要服务器作相应配置的，如果没做页面一刷新就会出问题
修改完成后在运行yarn build 就会发现 index.html里的文件路径都是以./开头
