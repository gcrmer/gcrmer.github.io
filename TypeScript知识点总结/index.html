关于TypeScript的一些知识

2022/05/21

TS和JS的区别

语法层面：TypeScript = JavaScript + Type（TS 是 JS 的超集）
执行环境层面：浏览器、Node.js 可以直接执行 JS，但不能执行 TS（Deno 可以执行 TS）
编译层面：TS 有编译阶段，JS 没有编译阶段（只有转译阶段和 lint 阶段）
编写层面：TS 更难写一点，但是类型更安全
文档层面：TS 的代码写出来就是文档，IDE 可以完美提示。JS 的提示主要靠 TS

type 和 interface 的区别

1.组合方式：interface 使用 extends 来实现继承，type 使用 | 来实现联合类型。
2.扩展方式：interface 可以重复声明用来扩展，type 一个类型只能声明一次
3.范围不同：type 适用于基本类型，interface 一般不行。
4.命名方式：interface 会创建新的类型名，type 只是创建类型别名，并没有新创建类型。
一般和对象相关的用interface，无关的用type。
