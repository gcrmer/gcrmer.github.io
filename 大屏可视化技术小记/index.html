大屏适配问题的思考

① 动态 rem 方案
背景:
我们希望在页面的大小发生变化时，页面元素的大小也能够发生变化，而且是成比例的进行缩放，这样就能够保证页面大小无论怎样变化，页面中的元素都能够恰当的显示，由此就产生了动态 rem 方案。

原理:
首先让 HTML 的 font-size 与 pageWidth(实际的页面宽度) 做相等的处理，即
1rem = pageWidth，同时为了元素在长度单位上书写方便，使用整数，一般将 1rem = pageWidth /100。
然后除字体使用 16px(字体比较小，缩放不会产生太大影响) ，其他的长度单位都时用 rem ，这样就能保证页面在布局上确定比例，在页面发生变化时，元素成比例的缩放。

② 大屏适配算法，确定大屏页面的尺寸

(W 设备 / H 设备) > (16/9) 即设备较宽的情况下
页面宽度 Wp = H 设备 x (16/9) ;

(W 设备 / H 设备) < (16/9) 即设备较窄的情况下
页面宽度 Wp= H 设备;

页面左右居中、上下居中、四周留白即可,然后在 head 里用 JS 设置 1rem = (Wp / 100)。

③ 设计稿中的元素在实际的大屏页面中的尺寸
公式:
Size 页面 / W 页面 = Size 设计 / W 设计;
W 页面 =100*rem;
Size 页面 = (Size 设计 / W 设计)100\ rem;

注:
Size 页面: 元素在大屏页面中的尺寸
W 页面:实际的大屏页面的宽度
Size 设计: 元素在设计稿中的尺寸
W 设计: 设计稿的宽度

例:
假设 div 在设计稿中的宽为 150px，设计稿宽度为 1920px，那么该 div 在大屏页面中的宽度为 150/1920100rem。

flex 布局的使用

指定一个容器为 flex 布局

定义容器的 6 个属性:

  • flex-direction
    • 决定主轴的方向,即项目的排列方向,它有4个值

    ①row:主轴为水平方向 起点在左端
    ②row-reverse:主轴为水平方向 起点在右端
    ③column:主轴为垂直方向 起点在上沿
    ④column:主轴为垂直方向 起点在下沿

  • flex-wrap
    • 决定项目是否换行,默认情况项目都排在一条轴线上,当一条轴线排不下时,该以何种方式换行呢?它将由以下3个值决定

    ①nowrap:默认值-不换行
    ②wrap:换行且第一行在上方
    ③wrap-reverse:换行且第一行在下方

  • flex-flow
    • 它是 flex-direction 与 flex-wrap 属性的简写形式 默认值是 row nowrap
  • justify-content
    • 定义项目在主轴上的对齐方式,它有以下5个值,假设主轴从左到右,那么

    ①flex-start:默认值-左对齐
    ②flex-end:右对齐
    ③center:居中对齐
    ④space-between:两端对齐,项目之间间隔相等
    ⑤space-around:每个项目两侧的间隔相等,所以项目之间的间隔是项目与边框的间隔的两倍

  • align-items
    • 定义项目在交叉轴上的对齐方式，它有5个值,对齐方式与交叉轴的方向有关，假设交叉轴从上到下,那么

    ①flex-start:交叉轴的起点对齐
    ②flex-end:交叉轴的终点对齐
    ③center:交叉轴的中点对齐
    ④baseline:项目第一行文字的基线对齐
    ⑤stretch:如果项目未设置高度或设为 auto,那么项目将会占满整个容器高度

  • align-content
    • 定义多根轴线的对齐方式,若项目仅有一根轴线,则属性不起作用,它有以下6个值

    ①flex-start:与交叉轴的起点对齐
    ②flex-end:与交叉轴的终点对齐
    ③center:与交叉轴的中点对齐
    ④space-between:与交叉轴的两端对齐,轴线之间的间隔相等
    ⑤space-around:每根轴线两侧的间隔都相等,轴线之间的间隔是轴线与边框之间间隔的两倍
    ⑥stretch:轴线占满整个交叉轴

定义项目的 6 个属性:

  • order
    • 定义项目的排列顺序,数值越小,位置越靠前
  • flex-grow
    • 定义项目的放大比例，默认为0，即有剩余空间也不放大，如果所有项目的flex-grow 属性都为1，则它们会等分剩余空间，如果其中一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间为其他项目的2倍。
  • flex-shrink
    • 定义项目的缩小比例，默认为1，即如果空间不足，该项目就会缩小。如果所有项目的flex-shrink属性都为1，当空间不足时，它们都将等比例缩小，如果一个项目的flex-shrink属性为0，其他项目都为1，若空间不足，前者不缩小，负值对该属性无效。
  • flex-basis
    • 定义在分配多余的空间之前，项目占据的主轴空间，默认值为auto，即项目的本来大小。
  • flex
    • 它是flex-grow，flex-shrink 和flex-basis的缩写，默认值是 0 1 auto，此属性有两个快捷值auto(1 1 auto)，none(0 0 auto)。
  • align-self
    • 它允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items，默认值auto，表示继承父元素的align-items属性，若没有父元素，则等同于stretch。
