HTTP 缓存的方案

HTTP1.1

缓存(强缓存)
响应头加上 max-age=3600，浏览器就会自动缓存一小时意思就是一个小时以内再次访问同样的 url，就直接不再发送请求,文件的特征值 Etag:ABC,这个就是强缓存.

内容协商
当强缓存过期后 浏览器向服务器发请求（ If-None-Match: ABC ），询问文件是否可以删除，服务器响应状态码如果为 304 表示继续使用无需删除，返回 200 表示删除，服务器会给最新的 JS 内容，这个过程就叫做内容协商（弱缓存）

HTTP1.0

缓存(强缓存):
与 HTTP1.1 不同的是，HTTP1.0 用时间来表示缓存的时长，同时给出上次文件修改的时间，在强缓存过期之后，浏览器通过文件修改时间向服务器询问是否需要更新。
Expires: Wed, 21 Oct 2020 02:30:00 GMT //缓存的有效期
Last-Modified: Wed, 21 Oct 2020 01:00:00 GMT //上次修改文件的时间

内容协商
If-Modified-Since: Wed, 21 Oct 2020 01:00:00 GMT 响应状态码：304 或 200 //服务器响应状态码为 304 表示文件无需更新，200 表示需要更新

如果用户电脑的时间不是标准时间就可能会造成时间错乱，导致缓存直接失效，而 HTTP1.1 直接规定缓存时长，就不存在这个问题。
